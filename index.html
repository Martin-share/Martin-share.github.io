为什么CLI击败了MCP
2026/3/27

Agent时代的路线之争：为什么CLI击败了MCP

核心观点

MCP的衰落不是因为技术不好，而是因为在Agent时代，简洁和高效战胜了规范和抽象。CLI的回归不是倒退，而是对AI本质的重新理解。

一、背景：从MCP的辉煌到转折

  • 2024年底：Anthropic推出MCP（Model Context Protocol），被誉为AI Agent的”USB接口”，各大厂商纷纷支持
  • 2025年12月：Anthropic将MCP捐赠给Linux基金会
  • 2026年初：风向突变，多个大厂开始放弃MCP转回CLI

关键转折事件

时间 事件 核心数据
2026年2月 微软发布Playwright CLI，建议替代Playwright MCP Token消耗从114,000降至27,000，节省76%
2026年3月11日 Perplexity CTO公开宣布放弃MCP MCP消耗72%的上下文窗口，认证机制笨重
2026年2月 OpenClaw创始人明确表示不支持MCP 认为MCP是”server to serve a server”的冗余抽象

二、MCP的三大天生缺陷

1. 线性上下文成本（致命伤）

  • 每个MCP工具都要占用Agent的上下文窗口（名称、描述、参数Schema）
  • 连接10个服务×5个工具=数千token在任务执行前就被烧掉
  • 仅加载一个Playwright MCP Server就占用上下文窗口的8%

2. 日常使用三座大山

  • 初始化地狱：配置文件错误、环境变量缺失、端口占用导致疯狂重启
  • 认证疲劳：每个工具需单独认证，且经常失效需重新授权
  • 权限粗糙：非黑即白的权限控制，无法细粒度控制读写权限

3. “builder多于user”的危险信号

  • 社区里讨论”如何构建MCP server”的人远多于”如何使用MCP”的人
  • Anthropic自己提出的”Code Execution with MCP”方案，Token使用量从150,000降至2,000，**节省了98.7%**——这实际上是在否定标准MCP工具调用

三、CLI复兴的五大优势

1. LLM的”母语”：命令行

  • 今天的LLM是在数十年Unix文档、Stack Overflow问答、GitHub代码上训练的
  • 执行gh pr view 123无需任何schema定义，LLM”刻在DNA里”的理解
  • MCP需要额外步骤：定义schema→让AI理解schema→才能工作

2. 调试透明度：输入一致，输出一致

  • CLI出错时，人类可以运行同样命令准确复现AI看到的内容
  • MCP工具仅存在于LLM对话内部，出错需翻找复杂的JSON传输日志

3. 可组合性：Unix哲学的胜利

  • grep | xargs、find | exec、git | diff等组合能力是CLI的天然优势
  • MCP要么把整个输出塞进上下文（成本高），要么把过滤逻辑做进server本身

4. 认证本来就能正常工作

  • CLI工具使用经过实战检验的认证流程：aws用profiles和SSO，gh用gh auth login
  • MCP在身份认证上过于”有主见”，反而增加复杂度

5. 无需管理额外进程

  • MCP server是进程，需要启动、保持运行、不能卡死
  • CLI工具只是磁盘上的二进制文件，没有后台进程，没有要管理的状态

四、本质洞察：Agent架构的范式转移

维度 MCP时代 CLI时代
Agent角色 “调度员”——通过标准接口代理，能力受限于Server提供的工具 “操作员”——直接在操作系统执行命令，能力上限=操作系统能力上限
能力边界 没有MCP Server就抓瞎 man页面就是工具文档，--help就是Schema
交互方式 JSON-RPC协议调用 自然语言→Shell→操作系统

关键结论：当AI足够智能，最好的”工具协议”就是自然语言+操作系统，不需要JSON-RPC、Schema、Server。

五、给开发者的建议

  1. 优先考虑CLI工具：更快、更便宜、更可靠
  2. 谨慎引入MCP：只在真正需要标准化的场景下使用
  3. 关注token效率：每个额外的抽象层都有成本
  4. 保持简单：不要为了”标准化”而牺牲实际价值
  5. 学好你的Shell：在AI时代，精通终端比以往任何时候都重要

结语

2000年代SOAP试图统一Web服务，但RESTful API以简洁性胜出；

2010年代追求”大一统”的服务网格，后来回归”足够简单”的解决方案。

MCP vs CLI的争论，是技术趋势钟摆的又一次摆动。正如文章标题所言：”MCP is dead. Long live the CLI.”——这不是倒退，而是前进；是回归本质，是去繁就简。
